Semi-functional with simulated data

Real buttons respond, simulated data looks real. No actual backend connection.

Tools: v0/Vercel Retool Streamlit

Your Role: "Does this work as I expect in real usage?"

üîê Constitutional Requirements (SACRED)

‚ö†Ô∏è These Rules Cannot Be Violated

The 5 CRs are SACRED and INVIOLABLE. Every piece of code must comply.

CR-001: Token Validity Check

System must verify token validity before ANY operation. No exceptions.

Impact: ALL API routes, ALL store actions, ALL renders

CR-002: Graceful Shutdown

Always allow graceful shutdown. 6-step sequence mandatory.

6 Steps: Stop ops ‚Üí Complete pending ‚Üí Close WS ‚Üí Clear memory ‚Üí Log ‚Üí Exit

CR-003: WHAT/WHY/HOW Errors

All errors must include WHAT happened, WHY, HOW to fix.

Format: No vague errors. 3-part structure always.

CR-004: 6 AM IST Token Expiry

Token expires daily at 6:00 AM IST (Zerodha policy).

Impact: TokenTimer, 30-min warnings, auto-terminate

CR-005: UXMI 7-States

Every component handles all 7 states: idle, hover, active, loading, success, error, disabled.

Matrix: 7 components √ó 7 states = 49 combinations
