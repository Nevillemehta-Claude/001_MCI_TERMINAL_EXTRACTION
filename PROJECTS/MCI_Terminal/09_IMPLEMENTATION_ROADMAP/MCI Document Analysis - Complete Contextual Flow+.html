Project Constitution

MCI (Mission Control Interface) is a decision-support trading dashboard for the Indian market. It operates under strict constitutional constraints that ensure it remains an information display system and NEVER executes trades autonomously.

  • Decision-Support ONLY - No buy/sell buttons
  • Descriptive AI - Never prescriptive recommendations
  • 6:00 AM IST Token Expiry - Sacred, inviolable timing
  • Equal Visual Weight - Bullish = Bearish presentation
  • 7-State UXMI - All interactive elements have full state coverage

üìú 5 Constitutional Requirements INVIOLABLE

‚ñº

Constitutional Requirements (CRs) are inviolable constraints that define the fundamental nature of the system. Violation of any CR is grounds for immediate rollback and remediation.

CR-001
Decision-Support, NOT Decision-Making

The system provides information to support human decisions but NEVER makes trading decisions.

  • ZERO trading signal generation
  • NO buy/sell buttons anywhere in the UI
  • Display information ONLY, never act on it
  • All execution happens externally in broker platforms
CR-002
Expose Contradictions, NEVER Resolve

When data sources conflict, the system displays all perspectives with equal weight.

  • Show conflicting states side-by-side
  • Equal visual weight for bullish and bearish signals
  • NEVER auto-fix or auto-reconcile conflicts
  • User interprets contradictions themselves
CR-003
Descriptive AI, NOT Prescriptive AI

AI components describe what IS, never what SHOULD BE.

  • Observations only: "Position is X"
  • NEVER recommend: "You should do Y"
  • No action verbs in AI output (buy, sell, hold)
  • Purely informational language
CR-004 SACRED
Token Lifecycle Accuracy - 6:00 AM IST Expiry

This constraint is SACRED and inviolable.

  • Kite tokens expire at exactly 6:00 AM IST (00:30 UTC)
  • Never approximate, never round
  • Countdown must be to-the-second accurate
  • System must gracefully handle expiry transition
  • No 24-hour countdown - FIXED DAILY TIME
// CR-004 Implementation (SACRED) const calculateNext6AMIST = () => { const expiry = new Date(); expiry.setUTCHours(0, 30, 0, 0); // 6:00 AM IST = 00:30 UTC if (new Date() >= expiry) { expiry.setDate(expiry.getDate() + 1); } return expiry.getTime(); };
CR-005
User Experience Micro-Interactions (UXMI)

All interactive elements must implement the 7-state UXMI specification.

  • 7 states: DEFAULT, HOVER, ACTIVE, FOCUS, DISABLED, LOADING, ERROR
  • Tooltips with 300ms hover delay
  • Error messages in WHAT/WHY/HOW format
  • Consistent animation curves (ease-out, 200ms)
  • 7 components: Tooltip, Button, Input, Spinner, ProgressBar, Toast, ErrorDisplay
